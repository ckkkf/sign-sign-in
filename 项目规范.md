# SignSignIn 项目规范

## Electron 目录分层

- `electron/src/main/api`：主进程接口层，只放真实 HTTP 请求。
- `electron/src/main/api/types`：接口请求、响应、上传策略等结构类型。
- `electron/src/main/services`：业务编排、任务流程、状态聚合，不直接散写原始 HTTP 请求。
- `electron/src/main/stores`：本地缓存、配置、草稿等持久化读写。
- `electron/src/preload`：只暴露 renderer 需要的 IPC API。
- `electron/src/renderer/src/pages`：页面级组件，负责组合业务区块。
- `electron/src/renderer/src/components`：可复用 UI 组件，按钮、状态块、弹窗等不要重复散写。
- `electron/src/renderer/src/composables`：页面状态、事件流程、轮询和副作用逻辑。
- `electron/src/shared`：主进程、预加载、渲染进程共享的类型、常量和纯函数。

## API 层规范

- 一个后端接口对应一个导出函数，例如 `login()`、`fetchDetail()`、`submitRecord()`。
- API 文件默认直接导出函数，不为无状态请求包 class/singleton。
- 只有存在真实实例状态、生命周期或可替换依赖时，才允许使用 class。
- URL、固定 payload、User-Agent、headers、params、body 放在对应接口函数内部，方便按请求定位。
- 不维护脱离函数的 endpoint map、payload map 或“万能请求方法”。
- 接口函数负责发请求和校验接口响应，不承载页面流程。
- 多接口组合流程放到 `services`，不要塞回 API 层。
- 通用 axios 实例放在 `electron/src/main/api/httpClient.ts`，同配置请求不要在每个 API 文件重复创建。
- 主进程发出的 HTTP 请求统一在 `httpClient.ts` 打日志，renderer DevTools 看不到主进程 axios 请求。
- 需要在 DevTools Network 面板观察的请求应由 renderer 发起；主进程只负责本地持久化和系统能力。

## 类型规范

- 接口专用结构放在 `electron/src/main/api/types`。
- 跨进程共享类型放在 `electron/src/shared/types.ts` 或同级共享文件。
- 不用 `any` 扩散到页面；确实来自第三方动态结构时，限定在 API/service 边界内。

## 代码风格

- 代码保持简洁、整齐、可读，避免过度设计和过度抽象。
- `if`、`throw`、`return`、`continue` 使用块级写法，不写成一行。
- 方法要写中文方法注释，关键业务步骤写中文行注释。
- 注释解释业务原因或协议要求，不写“给变量赋值”这类空注释。
- 文件内按请求流程或业务流程排列，避免来回跳转。
- 新增功能优先沿用现有组件、样式和服务分层。

## Renderer 规范

- 页面不要把所有 UI 和逻辑堆在 `App.vue`。
- 页面级组件只负责布局和组合，复杂状态放到 composable。
- 可复用按钮、导航、状态项、弹窗、横幅等抽为组件。
- 样式优先集中到现有 CSS 文件，避免在页面里散写 inline style。
- UI hover、选中、禁用等状态应由同一个组件统一实现。

## Electron 行为规范

- 主进程负责系统能力、网络请求、文件读写和托盘/菜单等能力。
- Renderer 不直接访问 Node 文件系统和主进程内部服务。
- IPC 名称保持清晰，按业务域分组，例如 `auth:*`、`jielong:*`。
- 右键菜单、系统代理、证书、抓包等平台能力统一放主进程。

## 校友邦与接龙功能规范

- 校友邦、接龙、桌面账号登录是三套独立登录态，不混用 token/session。
- 校友邦接口请求保持从 Python 版和抓包结果验证过的字段，不凭空增加参数。
- 接龙接口固定 payload 和 UA 放在具体接口函数内，便于逐接口核对。
- 抓包日志和业务日志按页面展示要求分区，不混在一起。
- 手动兜底入口删除后，不再新增类似临时输入框绕过主流程。

## 验证规范

修改 Electron 代码后至少执行：

```bash
cd electron
pnpm typecheck
pnpm test
pnpm build
```

UI 改动还要手动检查默认窗口高度、弹窗层级、导航 hover/选中状态、滚动条和横幅覆盖范围。
